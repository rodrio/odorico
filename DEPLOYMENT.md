# Odorico - Render Deployment Guide

This guide explains how to deploy the Odorico application to Render web services using Gunicorn.

## Prerequisites

1. **Render Account**: Sign up at [render.com](https://render.com)
2. **GitHub Repository**: Push your code to a GitHub repository
3. **API Keys**: Have your Google, OpenAI, and Anthropic API keys ready

## Files Added for Render Deployment

### 1. Procfile
```
web: gunicorn app:app --config gunicorn_config.py
```
- Tells Render how to start the application
- Uses Gunicorn with custom configuration

### 2. render.yaml
```yaml
services:
  - type: web
    name: odorico
    env: python
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn app:app --config gunicorn_config.py
    envVars:
      - key: PYTHON_VERSION
        value: 3.11.0
      - key: FLASK_ENV
        value: production
      - key: FLASK_DEBUG
        value: false
      - key: SECRET_KEY
        generateValue: true
      - key: GOOGLE_API_KEY
        sync: false
      - key: OPENAI_API_KEY
        sync: false
      - key: ANTHROPIC_API_KEY
        sync: false
```

### 3. gunicorn_config.py
- Custom Gunicorn configuration optimized for Render
- Handles worker processes, timeouts, and logging
- Environment-aware settings

### 4. .env.example
- Template for environment variables
- Reference for required API keys

## Code Changes Made

### App.py Updates
1. **Secret Key**: Now uses environment variable
   ```python
   app.secret_key = os.environ.get('SECRET_KEY', 'odorico-secret-key-change-in-production')
   ```

2. **Session Security**: Configurable via environment
   ```python
   app.config['SESSION_COOKIE_SECURE'] = os.environ.get('SESSION_COOKIE_SECURE', 'False').lower() == 'true'
   ```

3. **Production Mode**: Environment-based debug mode
   ```python
   if os.environ.get('FLASK_ENV') == 'production':
       app.run(host='0.0.0.0', port=int(os.environ.get('PORT', 5000)))
   ```

## Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Add Render deployment configuration"
git push origin main
```

### 2. Create Render Web Service

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click "New +" → "Web Service"
3. Connect your GitHub repository
4. Configure:
   - **Name**: odorico
   - **Environment**: Python 3
   - **Branch**: main
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app --config gunicorn_config.py`

### 3. Set Environment Variables

In Render Dashboard, add these environment variables:

**Required:**
- `GOOGLE_API_KEY`: Your Google Gemini API key
- `OPENAI_API_KEY`: Your OpenAI API key  
- `ANTHROPIC_API_KEY`: Your Anthropic API key

**Optional:**
- `SECRET_KEY`: Auto-generated by Render
- `FLASK_ENV`: production (auto-set)
- `FLASK_DEBUG`: false (auto-set)
- `SESSION_COOKIE_SECURE`: true (for HTTPS)
- `WEB_CONCURRENCY`: 3 (worker count)
- `LOG_LEVEL`: info

### 4. Deploy

1. Click "Create Web Service"
2. Render will automatically build and deploy
3. Monitor the build log for any issues

## Gunicorn Configuration

The `gunicorn_config.py` provides:

- **Workers**: 3 worker processes (configurable via `WEB_CONCURRENCY`)
- **Timeout**: 120 seconds for long API calls
- **Binding**: Automatically binds to Render's PORT
- **Logging**: Structured logging for monitoring
- **Graceful Shutdown**: Proper signal handling

## Performance Considerations

### Worker Count
- **Free Plan**: 3 workers (default)
- **Starter Plan**: Can increase to 4-6 workers
- **Adjust**: Set `WEB_CONCURRENCY` environment variable

### Timeout Settings
- **120 seconds**: Handles slow AI API responses
- **Can adjust**: Modify `timeout` in `gunicorn_config.py`

### Memory Usage
- **Monitor**: Check Render metrics
- **Optimize**: Adjust workers if memory issues occur

## Troubleshooting

### Common Issues

1. **Build Fails**:
   - Check requirements.txt for correct versions
   - Verify Python version compatibility

2. **Application Crashes**:
   - Check logs in Render dashboard
   - Verify API keys are set correctly
   - Ensure all environment variables are configured

3. **Slow Performance**:
   - Increase worker count
   - Check API key rate limits
   - Monitor memory usage

4. **Upload Issues**:
   - Ensure uploads directory permissions
   - Check file size limits

### Debug Mode

To enable debug logging:
```bash
# Set environment variable in Render
LOG_LEVEL=debug
```

### Health Checks

Render automatically checks if your app is running. The Gunicorn config ensures proper startup.

## Security Considerations

1. **API Keys**: Never commit to version control
2. **Secret Key**: Use Render's generated value
3. **HTTPS**: Enable `SESSION_COOKIE_SECURE=true`
4. **File Uploads**: Monitor uploaded files

## Monitoring

Render provides:
- **Metrics**: CPU, memory, response times
- **Logs**: Application and build logs
- **Alerts**: Error notifications
- **Health Checks**: Automatic uptime monitoring

## Scaling

To scale your application:
1. **Upgrade Plan**: Move to Starter or Pro plan
2. **Add Workers**: Increase `WEB_CONCURRENCY`
3. **Add Services**: Consider separate services for different features
4. **Database**: Add PostgreSQL if needed

## Local Testing

To test locally with Gunicorn:
```bash
# Install Gunicorn
pip install gunicorn

# Run with config
gunicorn app:app --config gunicorn_config.py

# Or directly
gunicorn app:app --bind 0.0.0.0:5000 --workers 3 --timeout 120
```
